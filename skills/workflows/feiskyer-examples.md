# Example Usage Scenarios

Use these as patterns, not rigid templates. Keep commands aligned with the installed AI coding tool's actual capabilities on the target machine.

## Read-Only Repository Analysis

## Read-Only Repository Analysis

Analyze a codebase structure without making changes. Use your search and read tools to explore the code structure, identify entry points, and assess refactoring risks.

## Safe Bug Fix

Fix a bug and run relevant tests. Work through the issue systematically: understand the problem, implement the fix, run the tests, and verify correctness.

## Batch Refactor

Update imports or patterns across the repository. Use search tools to find all affected locations, apply changes, and verify with the project's test suite.

## Security Report

Review the repository for security issues. Focus on concrete findings, exploitability, and mitigations. Return findings in a structured format.

## Resume a Session

Continue work from a previous session, restoring context and picking up where work left off.

## MCP-Assisted Investigation

Use available MCP servers to investigate issues (monitoring, databases, APIs, etc.). Leverage connected tools for deeper analysis.

## Fully Unattended Run in a Sandbox

Run autonomously in an isolated CI environment. Ensure the environment has appropriate safeguards and the task is fully self-contained.

## Read-Only Analysis

Explore the codebase with read-only tools only. Use search, grep, and file read operations to gather information without modifying anything.
